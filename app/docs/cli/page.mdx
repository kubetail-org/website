# Kubetail CLI Tool

The Kubetail CLI tool (`kubetail`) acts as an entry point into Kubetail on your desktop. Using the CLI tool you can start/stop the Kubetail Dashboard and perform other Kubetail-related operations on your cluster.

## Install/Upgrade

### Package Managers

#### macOS

##### Homebrew

To install `kubetail` with [homebrew](https://brew.sh/) use this command:

```sh
brew install kubetail
```

By default homebrew will place the executable in a directory that's already in to your path.

To upgrade `kubetail` just upgrade the package through homebrew:

```sh
brew update
brew upgrade
```

##### MacPorts

To install `kubetail` with [MacPorts](https://www.macports.org/) use this command:

```sh
sudo port install kubetail
```

To upgrade `kubetail` just upgrade the package through MacPorts:

```sh
sudo port upgrade kubetail
```

#### Linux

##### Ubuntu/Mint (apt)

To install `kubetail` from the [Kubetail PPA](https://launchpad.net/~kubetail/+archive/ubuntu/kubetail) on Ubuntu Launchpad, first install the `add-apt-repository` tool if you haven't already:

```sh
sudo apt update
sudo apt install software-properties-common
```

Then install `kubetail`:

```sh
sudo add-apt-repository ppa:kubetail/kubetail
sudo apt update
sudo apt install kubetail-cli
```

To upgrade `kubetail` just upgrade the package through apt:

```sh
sudo apt update
sudo apt upgrade kubetail-cli
```

##### Fedora/RHEL/CentOS (copr)

To install `kubetail` on Fedora/CentOS-Stream/RHEL (also Amazonlinux and Magea) with [copr](https://copr.fedorainfracloud.org/) use this command:

```sh
dnf copr enable kubetail/kubetail
dnf install kubetail
```

To upgrade `kubetail` just upgrade the package through dnf:

```sh
dnf upgrade kubetail
```

##### SUSE (zypper)

To install `kubetail` with openSUSE [zypper](https://documentation.suse.com/smart/systems-management/html/concept-zypper/index.html) use these commands:

```sh
zypper addrepo 'https://download.opensuse.org/repositories/home:/kubetail/$releasever/' kubetail
zypper refresh && zypper install kubetail-cli
```

To upgrade `kubetail` just upgrade the package through zypper:

```sh
zypper refresh
zypper update kubetail-cli
```

##### Alpine (apk)

To install `kubetail` on Alpine with [apk](https://wiki.alpinelinux.org/wiki/Alpine_Package_Keeper) you can use the [testing](https://wiki.alpinelinux.org/wiki/Repositories) repository:

```sh
apk add kubetail --repository=https://dl-cdn.alpinelinux.org/alpine/edge/testing
```

To upgrade `kubetail` just use the `upgrade` command:

```sh
apk upgrade kubetail
```

##### Arch Linux (AUR)

**With AUR helper (recommended)**

If you use an AUR helper like [yay](https://aur.archlinux.org/packages/yay) or [paru](https://aur.archlinux.org/packages/paru), install `kubetail` from [Arch Linux AUR](https://aur.archlinux.org/) with:

```sh
# yay
yay -S kubetail-cli

# paru
paru -S kubetail-cli
```

To upgrade `kubetail` just upgrade the package through your AUR helper:

```sh
# yay
yay -Syu kubetail-cli

# paru
paru -Syu kubetail-cli
```

**Without AUR helper**

To install `kubetail` manually from AUR:

```sh
git clone https://aur.archlinux.org/kubetail-cli.git
cd kubetail-cli && makepkg -si
```

To upgrade `kubetail` manually, repeat the process:

```sh
cd kubetail-cli
git pull
makepkg -si
```

##### Gentoo (GURU)

To install `kubetail` from the [Gentoo GURU](https://wiki.gentoo.org/wiki/Project:GURU/Information_for_End_Users) repository, first enable the GURU overlay if you haven't already:

```sh
emerge --ask app-eselect/eselect-repository
eselect repository enable guru
emerge --sync guru
```

Then install `kubetail`:

```sh
ACCEPT_KEYWORDS="~$(portageq envvar ARCH)" emerge dev-util/kubetail
```

To upgrade `kubetail` just sync and upgrade the package through emerge:

```sh
emerge --sync
ACCEPT_KEYWORDS="~$(portageq envvar ARCH)" emerge --update dev-util/kubetail
```

##### asdf

To install `kubetail` with [asdf](https://asdf-vm.com/) use this command:

To upgrade `kubetail` install the latest version and set it as global:

```sh
asdf install kubetail latest
```

### Shell script

To install `kubetail` with a shell script you can run this command:

```sh
curl -sS https://www.kubetail.com/install.sh | bash
```

This will download our [install.sh](/install.sh) script and run it in one step. The script will automatically detect your OS/Arch, download the appropriate binary file and place it in your `/usr/local/bin/` directory.

To upgrade `kubetail` just run the command again when a new version is available:

```sh
curl -sS https://www.kubetail.com/install.sh | bash
```

### Download Binary

First, download the binary for your OS/Arch (from the latest [release binaries](https://github.com/kubetail-org/kubetail/releases/latest)):

* [Darwin/amd64](https://github.com/kubetail-org/kubetail/releases/latest/download/kubetail-darwin-amd64)
* [Darwin/arm64](https://github.com/kubetail-org/kubetail/releases/latest/download/kubetail-darwin-arm64)
* [Linux/amd64](https://github.com/kubetail-org/kubetail/releases/latest/download/kubetail-linux-amd64)
* [Linux/arm64](https://github.com/kubetail-org/kubetail/releases/latest/download/kubetail-linux-arm64)
* [Windows/amd64](https://github.com/kubetail-org/kubetail/releases/latest/download/kubetail-windows-amd64)

Next, rename the file and make it executable:

```sh
mv <filename> kubetail
chmod a+x kubetail
```

To upgrade `kubetail` just repeat the steps above when a new version is available.

## Next Steps

To see the top-level commands just run `kubetail`:

```sh
kubetail
```

To start the Kubetail Dashboard use the `serve` sub-command:

```sh
kubetail serve
```

This command will start the dashboard server in your terminal and open [http://localhost:7500/](http://localhost:7500/) in your default browser.

To view your Kubernetes logs use the `logs` sub-command:

```sh
kubetail logs
```

To enable advanced functionality in the dashboard, you can install the Kubetail cluster resources (e.g. Cluster API, Cluster Agent) using this command:

```sh
kubetail cluster install
```
