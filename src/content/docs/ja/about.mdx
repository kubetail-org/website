Kubetail は Kubernetes 向けのログダッシュボードで、複数のログストリームを同時にリアルタイムで表示できます。Kubetail を使うと、リクエストがクラスター内でサービス間を移動する様子を簡単に追跡できます。Kubernetes でのロギングを楽しく、簡単で、プライベートなものにするために、開発者が開発者のために一から設計しました。

私たちが Kubetail の開発を始めたのは、Kubernetes 上でクラウドアプリを構築していたときに、`kubectl` や Kubernetes Dashboard でアプリをデバッグするのがあまりにも難しいと感じたからです。これらのツールは長寿命の単一コンテナにはうまく機能しましたが、複数のエフェメラルコンテナでは使いにくいものでした。そこで私たちは考えました。エフェメラルコンテナをネイティブに扱える、使いやすくログに特化した Kubernetes ダッシュボードを作るのは、どれほど難しいのだろうかと。

実際には想像以上に難しかったのですが、自分たちが使いたいツールを作るために必要なソフトウェアの部品はすでにそろっていることが分かりました。[Kubernetes API](https://kubernetes.io/docs/concepts/overview/kubernetes-api/)、[Go](https://go.dev/)、[React](https://react.dev/) をはじめ、多くのオープンソースツールを使って、データが手元を離れることなくログを確認できるオープンソースの Web ダッシュボードを作りました。このダッシュボードはデスクトップでもクラスター内でも動作します。

私たち自身も Kubetail を使っているので、Kubernetes 向けで最高のログツールにするために全力で取り組んでいます。フィードバックは大歓迎です（[hello@kubetail.com](mailto:hello@kubetail.com)）。ぜひ使ってみて、感想を聞かせてください。
