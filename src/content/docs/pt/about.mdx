Kubetail é um painel de logs para Kubernetes que permite visualizar vários fluxos de logs simultaneamente, em tempo real. Com o Kubetail, você pode acompanhar facilmente as requisições à medida que elas passam de serviço para serviço dentro do seu cluster. Ele foi projetado do zero por desenvolvedores, para desenvolvedores, para tornar o logging com Kubernetes divertido, simples e privado.

Começamos a trabalhar no Kubetail enquanto construíamos aplicativos em nuvem hospedados em Kubernetes e ficávamos frustrados com a dificuldade de depurar aplicações usando `kubectl` e o Kubernetes Dashboard. Embora essas ferramentas funcionassem bem para contêineres únicos e de longa duração, elas eram difíceis de usar com vários contêineres efêmeros. Isso nos levou a pensar: quão difícil seria criar um painel para Kubernetes, focado em logs, fácil de usar e capaz de lidar nativamente com contêineres efêmeros?

Foi definitivamente mais difícil do que imaginávamos, mas descobrimos que todas as ferramentas de software necessárias já existiam para construir a ferramenta que nós mesmos queríamos usar. Com a [API do Kubernetes](https://kubernetes.io/docs/concepts/overview/kubernetes-api/), [Go](https://go.dev/), [React](https://react.dev/) e um exército de outras ferramentas de código aberto, construímos um painel web de código aberto que roda no seu desktop ou dentro do seu cluster e permite visualizar seus logs sem que os dados saiam da sua posse.

Nós usamos o Kubetail no dia a dia, então pode ter certeza de que estamos trabalhando o máximo possível para torná-lo a melhor ferramenta de logs para Kubernetes. Adoramos receber feedback ([hello@kubetail.com](mailto:hello@kubetail.com)), então experimente e diga o que você acha.
