## Installation

L'outil CLI `kubetail` est le principal point d'entrée de Kubetail sur votre poste. Pour l'installer, vous pouvez utiliser votre gestionnaire de paquets préféré:

<Tabs syncKey="os">
    <TabItem label="macOS">
        **Homebrew**
        ```sh
        brew install kubetail
        ```

        **Krew**
        ```sh
        kubectl krew install kubetail
        ```

        **MacPorts**
        ```sh
        sudo port install kubetail
        ```

        **Nix (Flake)**
        ```sh
        nix profile add github:kubetail-org/kubetail-nix
        ```

        **Nix (Classic)**
        ```sh
        nix-env -i -f https://github.com/kubetail-org/kubetail-nix/archive/refs/heads/main.tar.gz
        ```

        **asdf**
        ```sh
        asdf plugin add kubetail https://github.com/kubetail-org/asdf-kubetail.git
        asdf install kubetail latest
        ```
    </TabItem>
    <TabItem label="Linux">
        **Homebrew**
        ```sh

        ```

        **Krew**
        ```sh
        kubectl krew install kubetail
        ```

        **Snap**
        ```sh

        ```

        **Ubuntu/Mint (apt)**
        ```sh
        sudo add-apt-repository ppa:kubetail/kubetail
        sudo apt update && sudo apt install kubetail-cli
        ```

        **Fedora/CentOS/RHEL/Amazonlinux/Mageia (copr)**

## Utilisation

Une fois installé, vous êtes prêt à commencer à tailer des logs. L'outil CLI `kubetail` utilise votre fichier kubeconfig pour l'authentification, vous aurez donc accès aux mêmes clusters que `kubectl`. Consultez la [référence de la CLI](/fr/docs/reference/cli) pour la liste complète des commandes.

* **Tableau de bord web**

    Lancez le tableau de bord web à l'aide de la commande [`serve`](/fr/docs/reference/cli#kubetail-serve) (il s'ouvrira sur [http://localhost:7500](http://localhost:7500)):

    ```sh
    kubetail serve
    ```

* **Terminal**

    Diffusez les logs directement dans votre terminal avec la commande [`logs`](/fr/docs/reference/cli#kubetail-logs):

    ```sh
    kubetail logs -f deployments/my-app
    ```

* **Configuration**

    Créez un fichier de configuration local à `~/.kubetail/config.yaml` à l'aide de la commande [`config init`](/fr/docs/reference/cli#kubetail-config):

    ```sh
    kubetail config init
    ```

* **Fonctionnalités avancées**

    Installez les ressources côté cluster à l'aide de la commande [`cluster install`](/fr/docs/reference/cli#kubetail-cluster) pour activer les fonctionnalités avancées (comme la recherche):

    ```sh
    kubetail cluster install
    ```
