컨테이너 내부에서 Kubetail CLI 도구(`kubetail`)를 실행하려면 Docker 이미지 [`kubetail-cli`](https://hub.docker.com/r/kubetail/kubetail-cli)를 사용할 수 있습니다. 이 이미지는 각 릴리스마다 Docker Hub와 GitHub Container Registry에 게시됩니다.

| Registry   | Image                               |
|------------|-------------------------------------|
| Docker Hub | `kubetail/kubetail-cli`             |
| GHCR       | `ghcr.io/kubetail-org/kubetail-cli` |

두 가지 변형이 제공됩니다.

| 태그 접미사 | 베이스 이미지 | 설명 |
|-------------|---------------|------|
| _(없음)_ | `scratch` | 가장 작은 크기의 최소 이미지 |
| `-alpine` | `alpine` | 셸과 표준 유틸리티 포함 |

Docker 내부에서 `kubetail`을 실행하면 Kubetail을 로컬 Docker 개발 워크플로에 통합할 수 있고, 클러스터 내부에서 로그를 디버그할 수도 있습니다.

<br />
<br />

<Aside type="tip">
Docker 내부에서 `kubetail`을 실행하는 일반적인 방법은 다음과 같습니다.

* [Docker Run](/ko/docs/guides/docker/docker-run)
* [Docker Compose](/ko/docs/guides/docker/docker-compose)
* [Kubernetes Pod](/ko/docs/guides/docker/kubernetes-pod)
</Aside>
