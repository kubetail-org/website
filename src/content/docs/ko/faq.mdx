## 일반

**1. Kubetail은 무엇이며 누구를 위한 도구인가요?**

Kubetail은 사용자가 자신의 Kubernetes 클러스터에서 여러 로그 스트림을 동시에 실시간으로 볼 수 있게 해 주는 Kubernetes용 로그 대시보드입니다.

**2. Kubetail은 어떤 Kubernetes 배포판이나 벤더를 지원하나요?**

Kubetail은 Kubernetes v1.18 이상을 사용하는 모든 Kubernetes 배포판과 벤더에서 작동합니다. Kubetail 사용 중 문제가 발생하면 [GitHub Issue](https://github.com/kubetail-org/kubetail/issues)를 만들어 주세요.

**3. Kubetail은 제 데스크톱에서 실행되나요, 아니면 클러스터 안에서 실행되나요?**

Kubetail은 [데스크톱](/ko/docs/guides/desktop/installation)에서 CLI를 통해 실행할 수도 있고, [클러스터](/ko/docs/guides/cluster/installation) 안에서 Dashboard Docker 이미지 배포 형태로 실행할 수도 있습니다.

**4. Kubetail은 얼마인가요?**

Kubetail은 완전히 무료입니다. 비용을 지불하지 않고 다운로드, 설치, 사용이 가능합니다. 또한 Kubetail은 사용자의 Kubernetes API를 직접 사용하고, 선택적으로 사용자의 클러스터 안에서 서비스를 실행하므로 도구가 사용하는 최소한의 리소스를 완전히 제어할 수 있습니다.

**5. Kubetail은 앞으로도 계속 무료인가요?**

네. 앞으로 유료 클라우드 서비스를 도입할 수는 있지만, Kubetail Dashboard, Kubetail CLI, 그리고 관련 도구는 언제나 무료로 유지됩니다. 우리는 가장 강력한 로깅 도구가 누구에게나 무료로 제공되어야 한다고 믿습니다.

**6. Kubetail은 메모리와 CPU를 얼마나 사용하나요?**

메모리와 CPU 사용량은 워크로드에 따라 달라지지만, 예를 들어 Kubetail [데모 사이트](https://demo.kubetail.com)를 위한 일반적인 배포는 다음과 같습니다.

| 구성 요소     | 메모리 | CPU    |
| :------------ | ------ | ------ |
| Dashboard     | ~21MB  | ~3.00m |
| Cluster API   | ~21MB  | ~3.00m |
| Cluster Agent | ~13MB  | ~4.00m |

**7. Kubetail의 소스 코드는 어디에서 찾을 수 있나요?**

Kubetail은 Apache 2.0 라이선스의 오픈 소스 프로젝트입니다. 저장소는 [https://github.com/kubetail-org/kubetail](https://github.com/kubetail-org/kubetail)에서 볼 수 있습니다.

**8. Kubetail은 앞으로도 계속 오픈 소스인가요?**

네. 우리는 클러스터 안에서 실행되는 모든 서드파티 도구는 오픈 소스여야 한다고 강하게 믿습니다. Kubetail을 완전한 오픈 소스로 유지하는 것은 이 프로젝트의 핵심 원칙입니다.

**9. 누가 Kubetail을 유지보수하나요?**

Kubetail은 GitHub 커뮤니티가 유지보수하는 오픈 소스 프로젝트입니다. Kubetail 핵심 코드에는 이 [contributors](https://github.com/kubetail-org/kubetail/graphs/contributors)가 기여했습니다. 또한 Helm 차트에는 이 [contributors](https://github.com/kubetail-org/helm-charts/graphs/contributors)가 기여했습니다. 현재 프로젝트의 리드 개발자는 Andres Morey ([@amorey](https://github.com/amorey/))입니다.

**10. Kubetail이 제 Kubernetes 클러스터에 접근하려면 어떤 자격 증명이나 권한이 필요한가요?**

데스크톱에서 실행되는 Kubetail은 로컬 kubeconfig 파일의 자격 증명을 사용해 Kubernetes API에 연결합니다. 클러스터에서 실행되는 Kubetail 리소스는 각 서비스 어카운트에 부여된 RBAC 권한을 사용합니다. 각 리소스에 필요한 구체적인 권한은 이 [YAML 매니페스트](https://github.com/kubetail-org/helm-charts/releases/latest/download/kubetail-clusterauth.yaml)에서 확인할 수 있습니다.

## 설치

**1. 데스크톱에서 Kubetail을 설치하고 접근하려면 어떻게 해야 하나요?**

[여기](/ko/docs/guides/desktop/installation)의 안내에 따라 데스크톱에서 Kubetail을 실행하세요.

**2. 클러스터에 Kubetail을 설치하고 접근하려면 어떻게 해야 하나요?**

[여기](/ko/docs/guides/cluster/installation)의 안내에 따라 클러스터 안에서 Kubetail을 실행하세요.

## 사용법

**1. Kubetail로 여러 클러스터의 로그를 볼 수 있나요?**

네. 데스크톱에서 Kubetail을 사용하면(즉, CLI를 통해), 로컬 kubeconfig 파일에 정의된 클러스터를 감지하고 대시보드 UI에서 클러스터 간 전환을 할 수 있습니다.

**2. Kubetail 사용에 대한 도움은 어떻게 받을 수 있나요?**

사용자를 돕는 일을 좋아합니다. 도움이 필요하면 다음 방법을 이용할 수 있습니다.

* [hello@kubetail.com](mailto:hello@kubetail.com)으로 이메일 보내기
* [GitHub Issue](https://github.com/kubetail-org/kubetail/issues) 만들기
* [Discord](https://discord.gg/CmsmWAVkvX)에서 대화하기
* [Slack](https://join.slack.com/t/kubetail/shared_invite/zt-2cq01cbm8-e1kbLT3EmcLPpHSeoFYm1w) 참여하기
* [tweet](https://x.com/kubetail) 보내기

**3. Kubetail에 어떻게 기여할 수 있나요?**

기여는 언제나 환영합니다. 방법은 다음과 같습니다.

1. [저장소](https://github.com/kubetail-org/kubetail)를 포크합니다
2. 변경 사항을 만듭니다
3. [GitHub Pull Request](https://github.com/kubetail-org/kubetail/pulls)를 제출합니다
