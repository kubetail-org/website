Kubetail 是一个面向 Kubernetes 的日志仪表板，可让您同时实时查看多个日志流。借助 Kubetail，您可以轻松跟踪请求在集群内部如何在各个服务之间流转。它从零开始由开发者为开发者打造，旨在让 Kubernetes 日志处理变得有趣、简单且私密。

我们最初开始开发 Kubetail，是因为在构建部署在 Kubernetes 上的云应用时，发现使用 `kubectl` 和 Kubernetes Dashboard 调试应用非常困难。虽然这些工具对于单个长期运行的容器效果不错，但在面对多个短生命周期容器时就很难使用了。这让我们开始思考：做一个专注于日志、易于使用、并且能够原生处理短生命周期容器的 Kubernetes 仪表板，到底会有多难？

事实证明，这确实比我们想象的更难，但我们也发现，构建我们自己想用的工具所需的软件能力其实都已经具备。借助 [Kubernetes API](https://kubernetes.io/docs/concepts/overview/kubernetes-api/)、[Go](https://go.dev/)、[React](https://react.dev/) 以及一系列其他开源工具，我们构建了一个开源 Web 仪表板。它既可以运行在桌面端，也可以运行在集群内部，并且让您在数据始终掌握在自己手中的前提下查看日志。

我们自己也在使用 Kubetail，因此请放心，我们会尽最大努力把它打造成 Kubernetes 上最好的日志工具。我们非常欢迎反馈（[hello@kubetail.com](mailto:hello@kubetail.com)），欢迎试用并告诉我们您的想法。
