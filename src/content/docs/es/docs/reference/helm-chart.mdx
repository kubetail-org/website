El chart de Helm de Kubetail despliega toda la pila de Kubetail en un cluster de Kubernetes. Se publica en nuestro repositorio de charts de Helm ubicado en [https://kubetail-org.github.io/helm-charts/](https://kubetail-org.github.io/helm-charts/), y el codigo fuente esta en el repositorio [kubetail-org/helm-charts](https://github.com/kubetail-org/helm-charts), dentro de `charts/kubetail`.

El chart despliega tres componentes:

| Component | Kind | Description |
|-----------|------|-------------|
| Dashboard | Deployment | Interfaz web y API backend |
| Cluster API | Deployment | API GraphQL para operaciones del cluster |
| Cluster Agent | DaemonSet | Agent por nodo que lee archivos de logs de contenedores |

## Instalacion

Agregue el repositorio Helm `kubetail`:

```sh
helm repo add kubetail https://kubetail-org.github.io/helm-charts/
```

Instale el chart en un namespace dedicado:

```sh
helm install kubetail kubetail/kubetail --namespace kubetail-system --create-namespace
```

Por defecto, el chart genera automaticamente los secretos necesarios (`KUBETAIL_DASHBOARD_SESSION_SECRET`) y los guarda en un Secret de Kubernetes para que se conserven en los upgrades.

## Upgrade

Obtenga el indice mas reciente del chart y luego actualice el release:

```sh
helm repo update kubetail
helm upgrade kubetail kubetail/kubetail --namespace kubetail-system
```

## Desinstalacion

```sh
helm uninstall kubetail --namespace kubetail-system
```
